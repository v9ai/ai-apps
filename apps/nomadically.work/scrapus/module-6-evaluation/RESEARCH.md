## Literature Review

### 1. End-to-End Evaluation of Multi-Stage NLP Pipelines

#### Error Propagation Analysis Framework

The Scrapus pipeline demonstrates a classic multi-stage NLP architecture where
errors can cascade through stages. Based on research by **Luan et al. (2018)**
[Multi-Task Identification of Entities, Relations, and Coreference for
Scientific Knowledge Graph Construction](https://doi.org/10.18653/v1/d18-1360),
cascading errors in pipeline architectures can be mitigated through joint
learning approaches.

**Error Propagation Matrix:**
```
Stage 1 (Crawling) -> Stage 2 (Extraction) -> Stage 3 (Matching) -> Stage 4 (Summarization)
    |                    |                    |                    |
Harvest rate       NER F1 (92.3%)     Matching F1 (0.88)    User satisfaction (92%)
~15% vs 5%         Precision (93.1%)  Precision (89.7%)    Factual accuracy (97%)
                   Recall (91.5%)     Recall (86.5%)
```

**Error Propagation Metrics:**
- **Cascade Error Rate (CER):** Probability that an error in stage i propagates to stage i+1. Measured: CER = 0.13.
- **Error Amplification Factor (EAF):** Ratio of downstream errors to upstream errors. Measured: EAF = 1.15x.
- **Pipeline Robustness Score:** 1 - (total propagated errors / total errors)

#### Multi-Stage Evaluation Protocol

Based on **Gatt & Krahmer (2018)** [Survey of the State of the Art in Natural
Language Generation](https://doi.org/10.1613/jair.5477):

1. **Stage-wise Isolation Testing:** Evaluate each component independently with gold-standard inputs
2. **End-to-end Integration Testing:** Measure system performance on complete workflow
3. **Error Attribution Analysis:** Use ablation studies to identify which stage contributes most to final errors

### 5. Limitations & Failure Modes

**Category 1 -- Data Quality:** Crawl frontier exhaustion, HTML parse errors,
entity ambiguity.

**Category 2 -- Model Performance:** Domain shift, concept drift, rare entity
types.

**Category 3 -- System Integration:** SQLite contention, memory pressure
(BERT + spaCy + LLM), queue synchronization.

**Category 4 -- User Experience:** Summary hallucination, ICP misalignment,
information overload.

## Automated Regression Test Implementation

### Architecture

The regression test suite is structured as a pytest session that runs the
full evaluation pipeline once and then asserts every quality gate. This avoids
re-running expensive model inference for each individual test.

```
test_regression.py
  |-- session fixture: run_full_evaluation()
  |     |-- load models (BERT, spaCy, XGBoost)
  |     |-- run NER on gold set (500 pages)
  |     |-- run matching on annotated leads (1,800)
  |     |-- run summarization on sample (100 reports)
  |     |-- return metrics dict
  |
  |-- test_ner_f1:          assert metrics["ner_f1"]          >= 0.90
  |-- test_lead_precision:  assert metrics["lead_precision"]  >= 0.85
  |-- test_lead_recall:     assert metrics["lead_recall"]     >= 0.80
  |-- test_report_accuracy: assert metrics["report_accuracy"] >= 0.93
  |-- test_crawl_harvest:   assert metrics["crawl_harvest"]   >= 0.10
```

### Threshold Rationale

| Metric           | Threshold | Source                                     |
|------------------|-----------|--------------------------------------------|
| NER F1 > 0.90    | 0.90      | 2.3pp margin below current 92.3%           |
| Lead P > 0.85    | 0.85      | 4.7pp margin below current 89.7%           |
| Lead R > 0.80    | 0.80      | 6.5pp margin below current 86.5%           |
| Report Acc > 0.93| 0.93      | 4pp margin below current 97%               |
| Harvest > 10%    | 0.10      | 5pp margin below current 15%               |

Margins are set to avoid false alarms from statistical noise while still
catching real degradation.

### Drift Detection Automation

```python
# run_monthly_drift_check.py
"""
Scheduled via cron: 0 9 1 * *  (1st of every month, 09:00)
"""
import json, sqlite3
from scrapus.evaluation import evaluate_on_gold_set
from scrapus.monitoring import detect_concept_drift

BASELINE_PATH = "baseline_metrics.json"
GOLD_SET_SIZE = 50

def main():
    with open(BASELINE_PATH) as f:
        baseline = json.load(f)

    current = evaluate_on_gold_set(n_samples=GOLD_SET_SIZE)

    # Log to metrics table
    conn = sqlite3.connect("scrapus_data/scrapus_metrics.db")
    conn.execute("""
        CREATE TABLE IF NOT EXISTS drift_checks (
            id INTEGER PRIMARY KEY,
            ts TEXT DEFAULT (datetime('now')),
            metrics_json TEXT,
            alert_fired INTEGER DEFAULT 0
        )
    """)

    try:
        detect_concept_drift(current, baseline, threshold_pp=5.0)
        conn.execute(
            "INSERT INTO drift_checks (metrics_json) VALUES (?)",
            (json.dumps(current),),
        )
    except RuntimeError as e:
        conn.execute(
            "INSERT INTO drift_checks (metrics_json, alert_fired) VALUES (?, 1)",
            (json.dumps(current),),
        )
        # Send alert (email, Slack, etc.)
        print(f"DRIFT ALERT: {e}")
        raise

    conn.commit()
    conn.close()

if __name__ == "__main__":
    main()
```

### Implementation Skeleton

```python
# dashboard.py
import streamlit as st
import sqlite3
import pandas as pd
import plotly.express as px

METRICS_DB = "scrapus_data/scrapus_metrics.db"
MAIN_DB = "scrapus_data/scrapus.db"

st.set_page_config(page_title="Scrapus Monitor", layout="wide")
st.title("Scrapus Pipeline Monitor")

# --- Section 1: Stage Latency ---
st.header("Stage Latency (last 7 days)")

conn = sqlite3.connect(METRICS_DB)
df_timing = pd.read_sql_query("""
    SELECT stage, elapsed_s, ts
    FROM stage_timing
    WHERE ts >= datetime('now', '-7 days')
    ORDER BY ts
""", conn)
conn.close()

if not df_timing.empty:
    df_timing["ts"] = pd.to_datetime(df_timing["ts"])
    df_timing["date"] = df_timing["ts"].dt.date

    # Compute daily median and P95
    daily = df_timing.groupby(["date", "stage"])["elapsed_s"].agg(
        median="median",
        p95=lambda x: x.quantile(0.95),
    ).reset_index()

    fig = px.line(daily, x="date", y="p95", color="stage",
                  title="P95 Latency by Stage")
    st.plotly_chart(fig, use_container_width=True)

# --- Section 2: Pipeline Throughput ---
st.header("Pipeline Throughput")

conn = sqlite3.connect(MAIN_DB)
df_pages = pd.read_sql_query("""
    SELECT date(crawl_timestamp) AS date, count(*) AS pages
    FROM crawled_pages
    WHERE crawl_timestamp >= datetime('now', '-30 days')
    GROUP BY date(crawl_timestamp)
""", conn)
df_leads = pd.read_sql_query("""
    SELECT date(created_at) AS date, count(*) AS leads
    FROM leads
    WHERE created_at >= datetime('now', '-30 days')
    GROUP BY date(created_at)
""", conn)
conn.close()

col1, col2 = st.columns(2)
with col1:
    if not df_pages.empty:
        st.bar_chart(df_pages.set_index("date")["pages"])
with col2:
    if not df_leads.empty:
        st.bar_chart(df_leads.set_index("date")["leads"])

# --- Section 3: Data Freshness ---
st.header("Data Freshness")

conn = sqlite3.connect(MAIN_DB)
df_fresh = pd.read_sql_query("""
    SELECT domain,
           CAST(julianday('now') - julianday(max(crawl_timestamp)) AS INTEGER) AS age_days
    FROM crawled_pages
    GROUP BY domain
    ORDER BY age_days DESC
    LIMIT 20
""", conn)
conn.close()

if not df_fresh.empty:
    def highlight_stale(val):
        return "background-color: #ffcccc" if val > 30 else ""
    st.dataframe(df_fresh.style.applymap(highlight_stale, subset=["age_days"]))

# --- Section 4: Quality Metrics (gauges) ---
st.header("Quality Metrics")

conn = sqlite3.connect(METRICS_DB)
row = conn.execute("""
    SELECT metrics_json FROM drift_checks ORDER BY ts DESC LIMIT 1
""").fetchone()
conn.close()

if row:
    import json
    metrics = json.loads(row[0])
    cols = st.columns(4)
    gauge_data = [
        ("NER F1", metrics.get("ner_f1", 0), 0.90),
        ("Lead Precision", metrics.get("lead_precision", 0), 0.85),
        ("Lead Recall", metrics.get("lead_recall", 0), 0.80),
        ("Report Accuracy", metrics.get("report_accuracy", 0), 0.93),
    ]
    for col, (name, value, threshold) in zip(cols, gauge_data):
        color = "green" if value >= threshold else "red"
        col.metric(name, f"{value:.1%}", delta=f"threshold: {threshold:.0%}")

# --- Section 5: Drift History ---
st.header("Drift History")

conn = sqlite3.connect(METRICS_DB)
df_drift = pd.read_sql_query("""
    SELECT ts, alert_fired, metrics_json
    FROM drift_checks
    ORDER BY ts
""", conn)
conn.close()

if not df_drift.empty:
    df_drift["status"] = df_drift["alert_fired"].map({0: "pass", 1: "alert"})
    fig = px.scatter(df_drift, x="ts", y="status", color="status",
                     color_discrete_map={"pass": "green", "alert": "red"},
                     title="Monthly Drift Checks")
    st.plotly_chart(fig, use_container_width=True)
```

## Recommendations for Enhanced Evaluation

1. **Implement Multi-Task Learning:** Following Luan et al. (2018), consider joint training of NER and relation extraction to reduce cascade errors

2. **Adopt Technology Readiness Levels (TRL):** As suggested by Lavin et al. (2022), establish TRL metrics for each pipeline component

3. **Enhanced Error Analysis:** Implement error attribution framework to quantify which stage contributes most to final errors

4. **Longitudinal Evaluation:** Track performance over time to measure concept drift and system degradation via the monitoring dashboard

5. **User-Centric Metrics:** Expand beyond accuracy to include:
   - Time-to-value (how quickly users get actionable leads)
   - Decision quality improvement (A/B testing with sales teams)
   - Return on investment (cost per qualified lead)

6. **Automated Gold Set Expansion:** Use active learning to select the most informative pages for annotation, growing the gold set from 500 to 2,000+ over time

7. **Adversarial Test Set:** Curate 50-100 deliberately misleading pages (SEO spam, honeypots) to quantify pipeline resilience
