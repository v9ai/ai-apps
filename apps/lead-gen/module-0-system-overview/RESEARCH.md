## Executive Summary

The Scrapus system is a local-first ML pipeline for B2B lead generation. It combines
RL-driven crawling, BERT NER extraction, Siamese entity resolution, XGBoost lead
scoring, and LLM summarization into a single-process Python application backed by
SQLite, LanceDB, and ChromaDB. No cloud infrastructure required beyond optional
OpenAI API access.

**Primary reference:** Kaplan, Seker & Yoruk (2025), "A review of AI-based business
lead generation: Scrapus as a case study," *Frontiers in AI*.

## 5. Storage Layer Analysis

### 5.1 SQLite for Graph Queries

Graph database emulation via adjacency tables:

- **Adjacency tables:** Traditional edge-list for relationships.
- **JSON columns:** `json_extract()` for flexible schema evolution.
- **FTS5:** Full-text search for entity name matching.
- **WAL mode:** Concurrent reads during writes; 1 writer + N readers.

Recursive CTEs handle shallow graph traversal (depth <= 2). For deeper traversals,
pre-compute transitive closures into a denormalized table.

**Research context:** Nguyen et al. (2019), "Machine Learning and Deep Learning
frameworks and libraries for large-scale data mining," discusses storage efficiency
for ML pipelines.

### 5.2 LanceDB for Vector Similarity

- **Apache Arrow format:** Columnar, zero-copy reads, efficient for vector ops.
- **On-disk ANN:** IVF_PQ index, no server process.
- **Four vector spaces:** entity (768-d), page (384-d), lead (512-d), replay (384-d).
- **Thread safety:** Single-writer; readers are safe. Write serialization via
  `asyncio.Lock`.

### 5.3 ChromaDB for Document Embeddings

- **Document memory:** Full page profiles with topic vectors.
- **BERTopic integration:** Stores cluster assignments and topic distributions.
- **Deduplication:** Cosine similarity search for content dedup.
- **HNSW tuning:** Per-collection `construction_ef` and `M` parameters.

### 6.1 SQLite Connection with Full Error Handling

```python
import sqlite3
import logging
from contextlib import contextmanager
from pathlib import Path

logger = logging.getLogger(__name__)

class DatabaseError(Exception):
    """Raised when database operations fail after retries."""

def open_connection(db_path: str, *, readonly: bool = False) -> sqlite3.Connection:
    """Open and configure a SQLite connection.

    Raises:
        FileNotFoundError: If db_path parent directory does not exist.
        DatabaseError: If PRAGMAs fail to apply.
    """
    path = Path(db_path)
    if not path.parent.exists():
        raise FileNotFoundError(f"Parent directory does not exist: {path.parent}")

    try:
        uri = f"file:{db_path}?mode=ro" if readonly else db_path
        conn = sqlite3.connect(uri, uri=readonly, timeout=10.0)
    except sqlite3.OperationalError as e:
        raise DatabaseError(f"Cannot open database: {e}") from e

    pragmas = [
        "PRAGMA journal_mode = WAL;",
        "PRAGMA synchronous = NORMAL;",
        "PRAGMA foreign_keys = ON;",
        "PRAGMA page_size = 4096;",
        "PRAGMA cache_size = -2000;",
        "PRAGMA mmap_size = 268435456;",
        "PRAGMA busy_timeout = 5000;",
        "PRAGMA wal_autocheckpoint = 1000;",
        "PRAGMA journal_size_limit = 67108864;",
    ]
    try:
        for pragma in pragmas:
            conn.execute(pragma)
        conn.row_factory = sqlite3.Row
    except sqlite3.OperationalError as e:
        conn.close()
        raise DatabaseError(f"PRAGMA configuration failed: {e}") from e

    logger.info("Opened SQLite connection: %s (readonly=%s)", db_path, readonly)
    return conn


@contextmanager
def transaction(conn: sqlite3.Connection):
    """Execute a block inside a transaction. Rolls back on exception."""
    try:
        yield conn
        conn.commit()
    except sqlite3.IntegrityError as e:
        conn.rollback()
        logger.warning("Integrity error, rolled back: %s", e)
        raise
    except sqlite3.OperationalError as e:
        conn.rollback()
        logger.error("Operational error, rolled back: %s", e)
        raise DatabaseError(f"Transaction failed: {e}") from e
    except Exception:
        conn.rollback()
        raise
```

### 6.2 LanceDB Entity Search with Error Handling

```python
import lancedb
import logging
from typing import Optional

logger = logging.getLogger(__name__)

class VectorSearchError(Exception):
    """Raised when vector search operations fail."""

def find_similar_entities(
    db: lancedb.DBConnection,
    embedding: list[float],
    entity_type: Optional[str] = None,
    min_confidence: float = 0.7,
    limit: int = 10,
) -> list[dict]:
    """Find entities similar to the given embedding.

    Args:
        db: LanceDB connection.
        embedding: Query vector (768-d float32).
        entity_type: Filter by type ("company", "person", "product").
        min_confidence: Minimum confidence threshold.
        limit: Max results to return.

    Returns:
        List of dicts with entity_id, similarity, confidence, metadata.

    Raises:
        VectorSearchError: If table does not exist or query fails.
    """
    try:
        tbl = db.open_table("entity_embeddings")
    except Exception as e:
        raise VectorSearchError(f"Cannot open entity_embeddings: {e}") from e

    try:
        query = tbl.search(embedding).metric("cosine").limit(limit * 2)
        if entity_type:
            query = query.where(f"entity_type = '{entity_type}'")
        if min_confidence > 0:
            query = query.where(f"confidence >= {min_confidence}")

        results = query.to_pandas()
    except Exception as e:
        raise VectorSearchError(f"ANN search failed: {e}") from e

    if results.empty:
        return []

    seen = set()
    output = []
    for _, row in results.iterrows():
        eid = row["entity_id"]
        if eid in seen:
            continue
        seen.add(eid)
        output.append({
            "entity_id": eid,
            "entity_type": row["entity_type"],
            "similarity": 1.0 - row["_distance"],
            "confidence": float(row["confidence"]),
        })
        if len(output) >= limit:
            break

    return output
```

### 6.3 ChromaDB Document Store with Error Handling

```python
import chromadb
import hashlib
import logging
from chromadb.config import Settings
from typing import Optional

logger = logging.getLogger(__name__)

class DocumentStoreError(Exception):
    """Raised when document store operations fail."""

def init_chromadb(path: str) -> chromadb.ClientAPI:
    """Initialize ChromaDB with all collections.

    Raises:
        DocumentStoreError: If ChromaDB fails to initialize.
    """
    try:
        client = chromadb.PersistentClient(
            path=path,
            settings=Settings(anonymized_telemetry=False, allow_reset=True),
        )
    except Exception as e:
        raise DocumentStoreError(f"ChromaDB init failed: {e}") from e

    collections = {
        "page_documents":    {"hnsw:space": "cosine", "hnsw:construction_ef": 200, "hnsw:M": 16},
        "company_documents": {"hnsw:space": "cosine", "hnsw:construction_ef": 400, "hnsw:M": 32},
        "topic_vectors":     {"hnsw:space": "cosine", "hnsw:construction_ef": 100, "hnsw:M": 8},
    }
    for name, meta in collections.items():
        try:
            client.get_or_create_collection(name=name, metadata=meta)
        except Exception as e:
            raise DocumentStoreError(f"Failed to create collection {name}: {e}") from e

    return client


def store_page(
    client: chromadb.ClientAPI,
    url: str,
    content: str,
    embedding: list[float],
    metadata: dict,
) -> str:
    """Store a page document. Returns the document ID.

    Raises:
        DocumentStoreError: If the insert fails.
        ValueError: If content is empty.
    """
    if not content.strip():
        raise ValueError("Cannot store empty content")

    content_hash = hashlib.sha256(content.encode()).hexdigest()[:16]
    url_hash = hashlib.sha256(url.encode()).hexdigest()[:8]
    doc_id = f"page_{url_hash}_{content_hash}"

    try:
        col = client.get_collection("page_documents")
        col.add(
            documents=[content[:10_000]],
            embeddings=[embedding],
            metadatas=[{**metadata, "url": url, "content_hash": content_hash}],
            ids=[doc_id],
        )
    except Exception as e:
        raise DocumentStoreError(f"Failed to store page {url}: {e}") from e

    logger.debug("Stored page %s as %s", url, doc_id)
    return doc_id
```

### 6.4 Async Pipeline Stage with Error Handling

```python
import asyncio
import logging
from concurrent.futures import ProcessPoolExecutor
from typing import Callable, Any

logger = logging.getLogger(__name__)

class PipelineStageError(Exception):
    """Raised when a pipeline stage fails fatally."""

async def run_stage(
    name: str,
    input_queue: asyncio.Queue,
    output_queue: asyncio.Queue,
    process_fn: Callable,
    pool: ProcessPoolExecutor,
    *,
    max_retries: int = 3,
    shutdown_event: asyncio.Event,
) -> None:
    """Run a pipeline stage that reads from input_queue and writes to output_queue.

    CPU-bound process_fn is executed in the ProcessPoolExecutor.
    Failed items are retried up to max_retries times, then logged and dropped.

    Args:
        name: Stage name for logging.
        input_queue: Source queue.
        output_queue: Destination queue.
        process_fn: CPU-bound function (must be picklable).
        pool: ProcessPoolExecutor for CPU offload.
        max_retries: Retry count per item.
        shutdown_event: Set this to stop the stage gracefully.
    """
    loop = asyncio.get_running_loop()

    while not shutdown_event.is_set():
        try:
            item = await asyncio.wait_for(input_queue.get(), timeout=1.0)
        except asyncio.TimeoutError:
            continue

        retries = 0
        while retries <= max_retries:
            try:
                result = await loop.run_in_executor(pool, process_fn, item)
                await output_queue.put(result)
                break
            except Exception as e:
                retries += 1
                if retries > max_retries:
                    logger.error(
                        "Stage %s: item dropped after %d retries: %s",
                        name, max_retries, e,
                    )
                else:
                    logger.warning(
                        "Stage %s: retry %d/%d: %s", name, retries, max_retries, e
                    )

    logger.info("Stage %s: shutdown complete", name)
```

## 7. Comparative Analysis

### Storage Alternatives

| Approach                      | Pros                                   | Cons                                 | Why Scrapus chose differently     |
|-------------------------------|----------------------------------------|--------------------------------------|-----------------------------------|
| Cloud managed (Neo4j, Pinecone) | Auto-scaling, HA, managed ops        | Cost, latency, privacy               | Privacy + offline requirement     |
