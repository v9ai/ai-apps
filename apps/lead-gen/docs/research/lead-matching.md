## Calibration Curve Analysis

Calibration measures how well predicted probabilities match observed
frequencies. A perfectly calibrated model predicts `P(qualified) = 0.80` for a
cohort where exactly 80% of companies are qualified.

### Method

1. Bin all test-set predictions into 10 equal-width bins (0.0-0.1, ...,
   0.9-1.0).
2. For each bin, compute the mean predicted probability and the fraction of
   actual positives.
3. Plot predicted vs. observed and compute the Expected Calibration Error
   (ECE).

### Interpretation

- The raw ensemble (without isotonic calibration) already shows reasonable

This checklist covers the operational requirements for enabling incremental

- [ ] **Label buffer table.** Create `CREATE TABLE label_buffer (id INTEGER
  PRIMARY KEY, company_id INTEGER, features BLOB, is_qualified INTEGER,
  ingested_at REAL)` in the scoring database.
- [ ] **Retrain trigger.** Cron job or row-count trigger that fires
  `OnlineLeadMatcher._retrain_window()` when `label_buffer` reaches 100 rows.
- [ ] **Model staging directory.** New model artefacts are written to
  `scrapus_data/models/staging/` before promotion to `scrapus_data/models/`.
- [ ] **Atomic model swap.** Use symlink rotation (`current -> v42/`) so
  in-flight scoring requests are not disrupted during model replacement.

### Validation Gates

- [ ] **Minimum performance gate.** After retraining, evaluate on the held-out
  test set (300 examples). Reject the new model if F1 < 0.85 or ROC-AUC <
  0.91.
- [ ] **Calibration gate.** Compute ECE on the test set. Reject if ECE > 0.05.
- [ ] **Shadow scoring.** Run the candidate model in parallel for 48 hours,
  logging predictions to `lead_explanations` with `model_version = 'shadow'`.
  Compare distributions before promotion.
- [ ] **Rollback trigger.** If F1 on the most recent 50 labeled examples
  drops below 80% of baseline, automatically rollback via `ModelRegistry`.

### Drift Detection

- [ ] **Feature drift monitor.** Track rolling mean and variance of each
  numeric feature. Alert when any feature's z-score exceeds 3.0 vs. the
  training distribution.
- [ ] **Prediction drift monitor.** Track the rolling mean of
  `ensemble_prob`. Alert when the 7-day mean deviates more than 0.10 from the
  training-set mean.
- [ ] **Label drift monitor.** Track the positive rate in the label buffer.
  Alert if it deviates more than 10 percentage points from the training
  positive rate (35%).

### Data Quality

- [ ] **Duplicate label check.** Reject duplicate `(company_id,
  feedback_source)` pairs in the label buffer.
- [ ] **Label delay tracking.** Log `ingested_at - scored_at` for each label.
  Alert if median delay exceeds 7 days (stale labels degrade retraining).
- [ ] **Feature completeness check.** Before retraining, verify that fewer
  than 20% of feature values in the buffer are missing (-1 sentinel). Pause
  retraining if exceeded.

### Logging & Audit

- [ ] **Retrain log table.** `CREATE TABLE retrain_log (id INTEGER PRIMARY
  KEY, model_version TEXT, triggered_at REAL, buffer_size INTEGER, train_f1
  REAL, train_roc_auc REAL, promoted INTEGER, rejection_reason TEXT)`.
- [ ] **Model lineage.** Each entry in `retrain_log` references the parent

## 6. Conformal Prediction for Uncertainty Quantification

### **Statistical Guarantees for Lead Scoring** (Based on 2023-2024 Advances)

**Scrapus Implementation:**
```python
# Conformal prediction for calibrated confidence intervals
class ConformalLeadScorer:
    def __init__(self, base_model, alpha=0.1):
        self.base_model = base_model
        self.alpha = alpha  # 90% confidence level
        self.calibration_scores = None

    def calibrate(self, X_calib, y_calib):
        # Get conformity scores on calibration set
        preds = self.base_model.predict_proba(X_calib)[:, 1]
        self.calibration_scores = np.abs(preds - y_calib)

        # Compute quantile for confidence level
        self.quantile = np.quantile(
            self.calibration_scores,
            np.ceil((1 - self.alpha) * (len(X_calib) + 1)) / len(X_calib)
        )

    def predict_with_confidence(self, X):
        point_pred = self.base_model.predict_proba(X)[:, 1]

        # Compute prediction intervals
        lower_bound = np.maximum(0, point_pred - self.quantile)
        upper_bound = np.minimum(1, point_pred + self.quantile)

        return {
            'point_estimate': point_pred,
            'confidence_interval': (lower_bound, upper_bound),
            'interval_width': upper_bound - lower_bound,
            'is_confident': (upper_bound - lower_bound) < 0.3  # Custom threshold
        }
```

**Key Benefits:**
1. **Statistical guarantees:** 90% of true scores fall within predicted intervals
2. **Adaptive thresholds:** Wider intervals for uncertain predictions
3. **Resource allocation:** Focus sales effort on high-confidence leads

## 1. SupCon Replacement for Siamese Networks

### Research Context & Novelty

**Prior findings** covered Siamese networks, but recent research shows **Supervised Contrastive Learning (SupCon)** provides superior representation learning with explicit label supervision. The key innovation is **label-aware contrastive loss** that pulls same-class embeddings together while pushing different-class embeddings apart.

**New Research (2024-2025):**
- **Bushiri et al. (2025)** demonstrate that semi-supervised contrastive learning enhances downstream prediction accuracy by 12-18% in business classification tasks [https://doi.org/10.1021/acs.jcim.4c00835]
- **Lopez Pombero (2025)** shows SupCon outperforms Siamese networks by 8-15% on tabular data for cognitive impairment diagnosis, particularly with imbalanced datasets [https://hdl.handle.net/20.500.14468/30314]

### Implementation: SupCon Training Loop

```python
import torch
import torch.nn.functional as F
from torch import nn

class SupConEncoder(nn.Module):
    """SupCon encoder with projection head for 128-dim embeddings"""
    def __init__(self, input_dim=512, hidden_dim=256, output_dim=128):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, output_dim)
        )
        self.projector = nn.Sequential(
            nn.Linear(output_dim, 64),
            nn.ReLU(),
            nn.Linear(64, 32)  # Projection for contrastive loss
        )

    def forward(self, x, return_projection=True):
        embedding = self.encoder(x)
        if return_projection:
            projection = self.projector(embedding)
            return embedding, F.normalize(projection, dim=1)
        return embedding

class SupConLoss(nn.Module):
    """Supervised Contrastive Loss with temperature scaling"""
    def __init__(self, temperature=0.07):
        super().__init__()
        self.temperature = temperature

    def forward(self, projections, labels):
        """
        projections: normalized embeddings [batch_size, projection_dim]
        labels: class labels [batch_size]
        """
        batch_size = projections.size(0)

        # Compute similarity matrix
        similarity = torch.matmul(projections, projections.T) / self.temperature

        # Create mask for positive pairs (same class)
        labels = labels.contiguous().view(-1, 1)
        mask = torch.eq(labels, labels.T).float()

        # Remove self-comparisons
        self_mask = torch.eye(batch_size, device=projections.device)
        mask = mask * (1 - self_mask)

        # Compute logits
        exp_sim = torch.exp(similarity)
        log_prob = similarity - torch.log(exp_sim.sum(1, keepdim=True))

        # Compute mean log-likelihood of positive pairs
        mean_log_prob_pos = (mask * log_prob).sum(1) / mask.sum(1)

        # Loss
        loss = -mean_log_prob_pos.mean()
        return loss

# Training configuration

### Implementation: FT-Transformer Architecture

```python
import torch
from torch import nn
import math

class FeatureTokenizer(nn.Module):
    """Tokenize tabular features for transformer input"""
    def __init__(self, num_numerical, categorical_cardinalities, d_model=128):
        super().__init__()

        # Numerical feature embedding
        self.numerical_projection = nn.Linear(num_numerical, d_model)

        # Categorical feature embeddings
        self.categorical_embeddings = nn.ModuleList([
            nn.Embedding(cardinality, d_model)
            for cardinality in categorical_cardinalities
        ])

        # Learnable [CLS] token
        self.cls_token = nn.Parameter(torch.randn(1, 1, d_model))

    def forward(self, numerical_features, categorical_features):
        """
        numerical_features: [batch_size, num_numerical]
        categorical_features: [batch_size, num_categorical]
        """
        batch_size = numerical_features.size(0)

        # Process numerical features
        numerical_tokens = self.numerical_projection(numerical_features).unsqueeze(1)

        # Process categorical features
        categorical_tokens = []
        for i, emb in enumerate(self.categorical_embeddings):
            cat_token = emb(categorical_features[:, i]).unsqueeze(1)
            categorical_tokens.append(cat_token)

        # Concatenate all tokens
        all_tokens = torch.cat([numerical_tokens] + categorical_tokens, dim=1)

        # Add CLS token
        cls_tokens = self.cls_token.expand(batch_size, -1, -1)
        tokens = torch.cat([cls_tokens, all_tokens], dim=1)

        return tokens

class FTTransformer(nn.Module):
    """FT-Transformer for lead classification"""
    def __init__(self, config):
        super().__init__()

        # Feature tokenizer
        self.tokenizer = FeatureTokenizer(
            num_numerical=config.num_numerical,
            categorical_cardinalities=config.categorical_cardinalities,
            d_model=config.d_model
        )

        # Position embeddings (learnable)
        self.pos_embedding = nn.Parameter(
            torch.randn(1, config.max_tokens, config.d_model)
        )

        # Transformer layers
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=config.d_model,
            nhead=config.n_heads,
            dim_feedforward=config.dim_feedforward,
            dropout=config.dropout,
            batch_first=True
        )
        self.transformer = nn.TransformerEncoder(
            encoder_layer, num_layers=config.num_layers
        )

        # Output head (CLS token only)
        self.output_head = nn.Sequential(
            nn.LayerNorm(config.d_model),
            nn.Linear(config.d_model, config.d_model // 2),
            nn.ReLU(),
            nn.Dropout(config.dropout),
            nn.Linear(config.d_model // 2, 1)
        )

    def forward(self, numerical_features, categorical_features):
        # Tokenize features
        tokens = self.tokenizer(numerical_features, categorical_features)

        # Add position embeddings
        tokens = tokens + self.pos_embedding[:, :tokens.size(1), :]

        # Transformer processing
        encoded = self.transformer(tokens)

        # Use CLS token for classification
        cls_output = encoded[:, 0, :]

        # Final prediction
        logits = self.output_head(cls_output)
        return torch.sigmoid(logits)

## 3. Conformal Prediction for Calibrated Confidence Intervals

### Research Context & Novelty

**Prior findings** lacked uncertainty quantification. **Conformal Prediction** provides distribution-free confidence intervals with guaranteed coverage, crucial for high-stakes business decisions.

**New Research (2024-2025):**

- **Sreenivasan et al. (2025)** show individualized diagnostic uncertainty quantification using conformal prediction enables reliable clinical decision-making [https://doi.org/10.1038/s41746-025-01616-z]
- **Shaker & Hullermeier (2025)** provide comprehensive analysis of random forest calibration, showing isotonic regression improves calibration by 15-25% [https://doi.org/10.1016/j.knosys.2025.114143]

### Implementation: Conformal Prediction Pipeline

```python
import numpy as np
from sklearn.isotonic import IsotonicRegression
from typing import Tuple, List

class ConformalPredictor:
    """Conformal prediction with split-conformal method"""
    def __init__(self, model, alpha=0.1, calibration_size=0.2):
        """
        model: base classifier with predict_proba method
        alpha: significance level (1 - coverage)
        calibration_size: proportion of data for calibration
        """
        self.model = model
        self.alpha = alpha
        self.calibration_size = calibration_size
        self.calibrator = IsotonicRegression(out_of_bounds='clip')
        self.calibration_scores = None
        self.quantile = None

    def fit(self, X_train, y_train):
        """Split data for training and calibration"""
        # Split indices
        n_cal = int(len(X_train) * self.calibration_size)
        cal_indices = np.random.choice(len(X_train), n_cal, replace=False)
        train_indices = np.setdiff1d(np.arange(len(X_train)), cal_indices)

        # Train on proper training set
        self.model.fit(X_train[train_indices], y_train[train_indices])

        # Calibrate on calibration set
        self._calibrate(X_train[cal_indices], y_train[cal_indices])

        return self

    def _calibrate(self, X_cal, y_cal):
        """Calibrate using conformal prediction"""
        # Get predicted probabilities
        probas = self.model.predict_proba(X_cal)[:, 1]

        # Compute non-conformity scores (1 - probability for true class)
        scores = []
        for i, (prob, true_label) in enumerate(zip(probas, y_cal)):
            if true_label == 1:
                score = 1 - prob  # Higher score for incorrect high confidence
            else:
                score = prob  # Higher score for incorrect low confidence
            scores.append(score)

        # Store calibration scores
        self.calibration_scores = np.array(scores)

        # Compute quantile for prediction sets
        n_cal = len(X_cal)
        self.quantile = np.quantile(
            self.calibration_scores,
            np.ceil((n_cal + 1) * (1 - self.alpha)) / n_cal
        )

    def predict_with_confidence(self, X_test) -> Tuple[np.ndarray, np.ndarray]:
        """
        Returns: (predictions, confidence_intervals)
        confidence_intervals: [lower_bound, upper_bound] for each prediction
        """
        # Get base predictions
        probas = self.model.predict_proba(X_test)[:, 1]

        # Apply conformal adjustment
        predictions = (probas >= 0.5).astype(int)

        # Compute confidence intervals
        confidence_intervals = []
        for prob in probas:
            # For class 1 prediction
            if prob >= 0.5:
                lower = max(0, prob - self.quantile)
                upper = min(1, prob + self.quantile)
            # For class 0 prediction
            else:
                lower = max(0, 1 - prob - self.quantile)
                upper = min(1, 1 - prob + self.quantile)
            confidence_intervals.append([lower, upper])

        return predictions, np.array(confidence_intervals)

    def get_prediction_sets(self, X_test, allow_empty=False) -> List[List[int]]:
        """
        Get prediction sets (multiple possible classes with coverage guarantee)
        """
        probas = self.model.predict_proba(X_test)
        prediction_sets = []

        for prob in probas:
            # Include class if 1 - probability <= quantile
            set_classes = []
            for class_idx, class_prob in enumerate(prob):
                score = 1 - class_prob
                if score <= self.quantile:
                    set_classes.append(class_idx)

            # If empty set not allowed, include most likely class
            if not allow_empty and not set_classes:
                set_classes = [np.argmax(prob)]

            prediction_sets.append(set_classes)

        return prediction_sets

class SiameseEncoder(nn.Module):
    def __init__(self, input_dim=768, hidden_dims=[512, 256], output_dim=128):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dims[0]),
            nn.BatchNorm1d(hidden_dims[0]),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dims[0], hidden_dims[1]),
            nn.BatchNorm1d(hidden_dims[1]),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dims[1], output_dim),
            nn.LayerNorm(output_dim),
        )

    def forward(self, x):
        return F.normalize(self.encoder(x), p=2, dim=1)
```

distance metrics for few-shot similarity tasks.

### Embedding Dimensions

### Training Loss

```python
class ContrastiveLoss(nn.Module):
    def __init__(self, margin=0.5):
        super().__init__()
        self.margin = margin

    def forward(self, output1, output2, label):
        d = F.pairwise_distance(output1, output2)
        loss = torch.mean(
            (1 - label) * torch.pow(d, 2)
            + label * torch.pow(torch.clamp(self.margin - d, min=0.0), 2)
        )
        return loss
```

### Location Embedding

Location strings are encoded via a learned 16-dimensional embedding trained
alongside the Siamese network:

```
location_string -> char trigrams -> Linear(300, 16)
```

The 300-dim input comes from hashing each trigram into a 300-bin vocabulary and
summing the one-hot vectors. The `Linear(300, 16)` layer is trained end-to-end
during contrastive learning and frozen at inference time.

                "num_sub_vectors": 32, "nprobe": 20}
```

## 7. Online Learning Framework

```python
class OnlineLeadMatcher:
    def __init__(self, base_models, buffer_size=1000):
        self.models = base_models
        self.label_buffer = []
        self.feature_buffer = []
        self.buffer_size = buffer_size

    def ingest_label(self, features, is_qualified):
        self.label_buffer.append(is_qualified)
        self.feature_buffer.append(features)
        if len(self.label_buffer) > self.buffer_size:
            self.label_buffer.pop(0)
            self.feature_buffer.pop(0)
        if len(self.label_buffer) >= 100:
            self._retrain_window()

    def _retrain_window(self):
        X = np.array(self.feature_buffer)
        y = np.array(self.label_buffer)
        for name, model in self.models.items():
            if hasattr(model, "partial_fit"):
                model.partial_fit(X, y)
            else:
                model.fit(X, y)
        self.label_buffer.clear()
        self.feature_buffer.clear()

    def detect_drift(self, baseline_f1):
        if len(self.label_buffer) < 50:
            return False
        recent_f1 = f1_score(self.label_buffer[-50:],
                             [self.predict(f) for f in self.feature_buffer[-50:]])
        return recent_f1 < baseline_f1 * 0.80
```

## Novel Approaches

# Novel Insights -- Module 4: Lead Matching

## Surprising Findings

- **128-dim is likely oversized for 2,400 training samples.** With only 2,400 labeled examples, a 128-dim Siamese embedding space is underconstrained. The CoSiNES paper (Yuan et al., 2023) achieved 92.3% accuracy with a 32-dim projection head on comparable entity-matching tasks. The projection head trick -- train at 128-dim, project down to 32 for contrastive loss, discard the projector at inference -- means you get the representational capacity during training without the curse of dimensionality at retrieval time. Most teams never try shrinking the inference embedding because it feels wrong, but the projector exists precisely to decouple training geometry from retrieval geometry.

- **The ensemble barely helps.** The README shows F1 = 0.883 for the 50/25/25 ensemble. Run XGBoost alone at the same threshold and the gap is almost certainly < 0.01 F1. The LogReg and RF components contribute smoothing, not signal. The real value of the ensemble is calibration -- averaging overconfident XGBoost predictions (0.1-0.4 range) with underconfident LogReg (0.7-0.9 range) gives you ECE 0.028 for free. The moment you add isotonic calibration as a post-processing step (which is already implemented but not wired in), the ensemble's calibration advantage vanishes and you're paying 3x inference cost for nothing.

- **Calibration is broken at the decision boundary.** The calibration table shows the 0.8-0.9 bin has predicted=0.84, observed=0.87 -- a +3pp gap right where the 0.85 threshold lives. This means the model is systematically underconfident near the cutoff. Leads scored 0.83-0.85 that get rejected are actually converting at ~0.87. You're losing ~3% recall for free. Isotonic calibration fixes this, but nobody noticed because ECE averages over all bins and 0.028 looks fine.

- **The 35% positive rate is suspiciously balanced.** Real B2B lead qualification datasets have 5-15% positive rates. A 35% rate suggests either aggressive pre-filtering before labeling (which means the hard negatives are missing) or labeler bias toward "yes." This matters because the model has never seen the distribution it will face in production. The Janssens B2Boost paper (2022) explicitly warns that profit-driven models trained on curated datasets collapse when deployed on raw pipelines.

## Unconventional Techniques

- **ICP as a generative model, not a filter.** Instead of encoding the ICP as a single vector and measuring cosine distance, treat the ICP as a conditional distribution: P(company_features | qualified=1). Train a small normalizing flow or VAE on qualified companies. At inference, score candidates by log-likelihood under the ICP distribution. This captures multimodal ICPs (e.g., "we want either large enterprises in healthcare OR small AI startups") that a single centroid vector fundamentally cannot represent. A mixture of 3-5 Gaussians in the embedding space is a cheap approximation.

- **Lead scoring as a recommendation problem.** The Scrapus architecture treats scoring as binary classification, but it is structurally identical to a recommendation problem: match items (companies) to a user (sales rep + ICP). Reframe it as collaborative filtering. If you have multiple ICPs or multiple sales reps with different win patterns, matrix factorization over (rep, company, outcome) triples captures per-rep preferences that a single global model misses entirely. The Xiao et al. (2025) cost-sensitive learning paper gestures at this with per-example costs, but the full recommender framing is more powerful.

- **Score via LLM chain-of-thought, then distill.** Use an LLM to produce chain-of-thought qualification rationales for a sample of leads: "This company is a fit because they operate in AI security (ICP match), have 200 employees (size match), but their product focus is consumer not enterprise (partial miss). Score: 0.72." Extract the intermediate reasoning features (ICP match flags, mismatch reasons) as structured labels. Train a small model on these synthetic structured labels. You get the LLM's nuanced reasoning distilled into a fast classifier, plus the reasoning traces become your explainability layer -- better than SHAP values because they're in natural language and capture interaction effects.

- **Contrastive augmentation that actually matters for tabular data.** The DEEP_RESEARCH prescribes Gaussian noise, feature masking, and column shuffling as augmentations. These are borrowed from vision and mostly useless for tabular. The augmentation that actually works: swap a feature value with a value from another company in the same industry but opposite class. This forces the encoder to learn which features are discriminative within-industry rather than just memorizing "healthcare companies are good." Bushiri et al. (2025) report 12-18% improvement, but only with semantically meaningful augmentations, not random noise.

## Hidden Gems (Papers)

- **Janssens et al. (2022) "B2Boost"** -- This is not just another XGBoost paper. It introduces instance-dependent profit-driven loss functions for B2B churn. The key insight: a false negative on a $500K account costs 100x more than on a $5K account. Scrapus treats all misclassifications equally. Plugging deal size into the loss function (available from the enrichment pipeline) would let the model trade precision on small accounts for recall on large ones. Nobody does this because sklearn doesn't support it out of the box -- you need custom XGBoost objectives.

- **Hullermeier & Waegeman (2021) "Aleatoric and Epistemic Uncertainty"** -- Listed in the references but completely ignored in the architecture. The distinction matters: aleatoric uncertainty (this company genuinely could go either way) should trigger human review. Epistemic uncertainty (we've never seen a company like this) should trigger data collection. The conformal prediction proposal conflates the two. A proper decomposition uses MC dropout or deep ensembles for epistemic, and heteroscedastic regression for aleatoric. Operationally: high epistemic leads go to enrichment, high aleatoric leads go to sales for manual judgment.

- **Curiskis et al. (2023) "Customer Lifetime Value in B2B SaaS"** -- Hidden in the references, never discussed. The insight: scoring leads by conversion probability is wrong. Score by expected revenue = P(convert) x E(deal_size | convert) x E(retention | convert). A 60% probability lead worth $200K/year is better than a 90% probability lead worth $10K/year. The Scrapus pipeline has funding_amount as a feature but never uses it as a target modifier. This requires reframing from classification to regression on expected value.

- **Bidgoli et al. (2024) "Predicting Startup Success"** -- Uses network topology features (founder connections, investor overlap) that outperform firmographic features for early-stage companies where traditional signals (revenue, employee count) are noise. Scrapus has `social_presence` as a sum of booleans, but the graph structure of who-links-to-whom on LinkedIn is orders of magnitude more predictive than whether a LinkedIn page exists.

## Contrarian Takes

- **FT-Transformer will not beat XGBoost on this dataset.** The research shows 7-12% improvement on "heterogeneous tabular data with mixed categorical/numerical features." Scrapus has 8 numerical features, 4 binary flags, and one 47-category one-hot. This is not high-cardinality. This is not heterogeneous. XGBoost with 200 trees on 2,400 samples is already near the ceiling. FT-Transformer needs 10x more data to learn meaningful attention patterns. The Sodnomdavaa fraud detection paper (2025) shows TabNet beats XGBoost by 0.4% AUC -- not 7-12%. The big improvements come only when you have 50+ categorical features with cardinality >100. Adding FT-Transformer to the ensemble will add complexity, training time, and approximately zero F1.

- **The knowledge graph is a research trap.** Arora (2025) reports +7.3pp AUC from graph features in "enterprise SaaS context" -- meaning they had rich CRM data with explicit deal relationships, board connections, and technology purchase histories. Scrapus has scraped web pages. Building a company-person-technology graph from scraped data produces a sparse, noisy graph where centrality metrics are artifacts of scraping coverage, not business reality. The effort-to-signal ratio is brutal. Better to spend the same engineering time on intent signals (job postings, technology adoption, funding events).

- **Simpler models beat deep learning here, and that's fine.** The entire DEEP_RESEARCH document pushes toward foundation models, GNNs, and multi-modal transformers. For 2,400 training samples with 15-20 features, a well-tuned LightGBM will match or beat any deep architecture. The Imani et al. (2025) systematic review confirms this: LightGBM dominates on B2B datasets under 100K samples. The path to better performance is not architectural complexity -- it's more labeled data, better features, and proper calibration. The SupCon encoder swap is the one exception because it improves the embedding stage, which operates on text (where neural models genuinely help), not the tabular classification stage.

- **Online learning with a 100-row buffer will oscillate.** The proposed retrain trigger fires at 100 new labels. With a 35% positive rate, that's 35 positives and 65 negatives. XGBoost's performance on 100 incremental samples will be noisy. The validation gate (F1 >= 0.85) will reject most retrains, wasting compute. And when it does accept one, the model shifts toward recent distribution, which may be a temporary fluctuation. Better: accumulate labels continuously, retrain monthly with the full historical dataset, use exponential decay weighting on older samples. The label buffer pattern comes from online ad tech where you get millions of labels per day -- it does not transfer to B2B where you get 100 labels per month.

## Wild Ideas

- **Temporal scoring: predict WHEN to contact, not just WHO.** Companies go through buying cycles. A company that just raised Series B is in a 3-month window where they staff up and adopt new tools. A company that just went through layoffs is in a 6-month freeze. Encode temporal signals (funding date, job posting velocity, leadership changes) into a time-to-purchase model. Output: "Contact in 6 weeks" is more valuable than "Score: 0.87." This requires event stream data (Crunchbase, LinkedIn job postings) rather than static snapshots, but the enrichment pipeline already has timestamps.

- **Causal scoring: what intervention maximizes conversion.** The current model answers "will this company convert?" The useful question is "will this company convert IF we do X?" where X is {cold email, warm intro, conference meeting, product demo, case study}. This is a causal inference problem (treatment effect estimation). Use the historical labeled data to estimate conditional average treatment effects per channel. A company scored 0.6 via cold email might be 0.9 via warm intro. This requires logging the outreach channel alongside conversion labels, which the `feedback_source` field in the label buffer could capture if repurposed.

- **Company narrative embeddings from earnings calls and blog posts.** Instead of encoding company descriptions (static, marketing-speak), embed the narrative arc: what the company talks about over time, how their messaging shifts, what topics they emphasize quarter over quarter. A company whose blog posts shift from "product features" to "enterprise security compliance" is signaling upmarket movement -- a leading indicator invisible to static embeddings. Use a temporal document embedding model (not just averaging BERT vectors) that captures trajectory, not position.

- **Adversarial ICP refinement.** Use the model's false positives to refine the ICP. Companies scored >0.85 that sales rejects are the most informative signal in the pipeline. Cluster false positives and surface the distinguishing features to the ICP owner: "Your ICP says AI security companies, but you keep rejecting companies focused on network security vs. application security. Should we split the ICP?" This closes the loop between model output and ICP definition, turning the scoring pipeline into an ICP discovery tool.

- **Scoring as a bandit problem.** Each lead scored is a decision under uncertainty. The threshold (0.85) is a static policy. Replace it with a contextual bandit that adapts the threshold based on current pipeline fullness, sales team capacity, and recent conversion rates. When the pipeline is empty, lower the threshold to increase volume. When the pipeline is full, raise it to increase quality. Thompson sampling over the threshold parameter gives you exploration (occasionally letting marginal leads through to measure their conversion) for free.

## 1. Contrastive Learning Beyond Siamese Networks

### **CoSiNES: Contrastive Siamese Network for Entity Standardization** (Yuan et al., 2023)

# Replace traditional Siamese with contrastive pretraining
class ContrastiveICPEncoder:
    def __init__(self):
        self.backbone = SentenceTransformer('all-mpnet-base-v2')
        self.projection_head = nn.Linear(768, 256)  # Contrastive projection

    def pretrain_contrastive(self, company_corpus):
        # Cross-company contrastive learning
        # Positive pairs: same industry, different companies
        # Negative pairs: different industries
        loss = NTXentLoss(temperature=0.07)

    def encode_icp(self, attributes):
        # Multi-attribute fusion with attention
        embeddings = {
            'industry': self.backbone.encode(attributes['industry']),
            'size': self.backbone.encode(str(attributes['size'])),
            'location': self.backbone.encode(attributes['location'])
        }
        # Learn attention weights per attribute
        return self.fuse_with_attention(embeddings)
```

# Conformal prediction for calibrated confidence intervals
class ConformalLeadScorer:
    def __init__(self, base_model, alpha=0.1):
        self.base_model = base_model
        self.alpha = alpha  # 90% confidence level
        self.calibration_scores = None

    def calibrate(self, X_calib, y_calib):
        # Get conformity scores on calibration set
        preds = self.base_model.predict_proba(X_calib)[:, 1]
        self.calibration_scores = np.abs(preds - y_calib)

        # Compute quantile for confidence level
        self.quantile = np.quantile(
            self.calibration_scores,
            np.ceil((1 - self.alpha) * (len(X_calib) + 1)) / len(X_calib)
        )

    def predict_with_confidence(self, X):
        point_pred = self.base_model.predict_proba(X)[:, 1]

        # Compute prediction intervals
        lower_bound = np.maximum(0, point_pred - self.quantile)
        upper_bound = np.minimum(1, point_pred + self.quantile)

        return {
            'point_estimate': point_pred,
            'confidence_interval': (lower_bound, upper_bound),
            'interval_width': upper_bound - lower_bound,
            'is_confident': (upper_bound - lower_bound) < 0.3  # Custom threshold

class ContrastiveLoss(nn.Module):

        super().__init__()
        self.margin = margin

    def forward(self, output1, output2, label):
        euclidean_distance = F.pairwise_distance(output1, output2)
        loss = torch.mean((1-label) * torch.pow(euclidean_distance, 2) +

The key innovation is **label-aware contrastive loss** that pulls same-class embeddings together while pushing different-class embeddings apart.

**New Research (2024-2025):**
- **Bushiri et al. (2025)** demonstrate that semi-supervised contrastive learning enhances downstream prediction accuracy by 12-18% in business classification tasks [https://doi.org/10.1021/acs.jcim.4c00835]

import torch
import torch.nn.functional as F
from torch import nn

class SupConEncoder(nn.Module):
    """SupCon encoder with projection head for 128-dim embeddings"""
    def __init__(self, input_dim=512, hidden_dim=256, output_dim=128):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, output_dim)
        )
        self.projector = nn.Sequential(
            nn.Linear(output_dim, 64),
            nn.ReLU(),
            nn.Linear(64, 32)  # Projection for contrastive loss
        )

    def forward(self, x, return_projection=True):
        embedding = self.encoder(x)
        if return_projection:
            projection = self.projector(embedding)
            return embedding, F.normalize(projection, dim=1)
        return embedding

class SupConLoss(nn.Module):
    """Supervised Contrastive Loss with temperature scaling"""
    def __init__(self, temperature=0.07):
        super().__init__()
        self.temperature = temperature

    def forward(self, projections, labels):
        """
        projections: normalized embeddings [batch_size, projection_dim]
        labels: class labels [batch_size]
        """
        batch_size = projections.size(0)

        # Compute similarity matrix
        similarity = torch.matmul(projections, projections.T) / self.temperature

        # Create mask for positive pairs (same class)
        labels = labels.contiguous().view(-1, 1)
        mask = torch.eq(labels, labels.T).float()

        # Remove self-comparisons
        self_mask = torch.eye(batch_size, device=projections.device)
        mask = mask * (1 - self_mask)

        # Compute logits
        exp_sim = torch.exp(similarity)
        log_prob = similarity - torch.log(exp_sim.sum(1, keepdim=True))

        # Compute mean log-likelihood of positive pairs
        mean_log_prob_pos = (mask * log_prob).sum(1) / mask.sum(1)

        # Loss
        loss = -mean_log_prob_pos.mean()
        return loss

```python
import torch
from torch import nn
import math

class FeatureTokenizer(nn.Module):
    """Tokenize tabular features for transformer input"""
    def __init__(self, num_numerical, categorical_cardinalities, d_model=128):
        super().__init__()

        # Numerical feature embedding
        self.numerical_projection = nn.Linear(num_numerical, d_model)

        # Categorical feature embeddings
        self.categorical_embeddings = nn.ModuleList([
            nn.Embedding(cardinality, d_model)
            for cardinality in categorical_cardinalities
        ])

        # Learnable [CLS] token
        self.cls_token = nn.Parameter(torch.randn(1, 1, d_model))

    def forward(self, numerical_features, categorical_features):
        """
        numerical_features: [batch_size, num_numerical]
        categorical_features: [batch_size, num_categorical]
        """
        batch_size = numerical_features.size(0)

        # Process numerical features
        numerical_tokens = self.numerical_projection(numerical_features).unsqueeze(1)

        # Process categorical features
        categorical_tokens = []
        for i, emb in enumerate(self.categorical_embeddings):
            cat_token = emb(categorical_features[:, i]).unsqueeze(1)
            categorical_tokens.append(cat_token)

        # Concatenate all tokens
        all_tokens = torch.cat([numerical_tokens] + categorical_tokens, dim=1)

        # Add CLS token
        cls_tokens = self.cls_token.expand(batch_size, -1, -1)
        tokens = torch.cat([cls_tokens, all_tokens], dim=1)

        return tokens

class FTTransformer(nn.Module):
    """FT-Transformer for lead classification"""
    def __init__(self, config):
        super().__init__()

        # Feature tokenizer
        self.tokenizer = FeatureTokenizer(
            num_numerical=config.num_numerical,
            categorical_cardinalities=config.categorical_cardinalities,
            d_model=config.d_model
        )

        # Position embeddings (learnable)
        self.pos_embedding = nn.Parameter(
            torch.randn(1, config.max_tokens, config.d_model)
        )

        # Transformer layers
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=config.d_model,
            nhead=config.n_heads,
            dim_feedforward=config.dim_feedforward,
            dropout=config.dropout,
            batch_first=True
        )
        self.transformer = nn.TransformerEncoder(
            encoder_layer, num_layers=config.num_layers
        )

        # Output head (CLS token only)
        self.output_head = nn.Sequential(
            nn.LayerNorm(config.d_model),
            nn.Linear(config.d_model, config.d_model // 2),
            nn.ReLU(),
            nn.Dropout(config.dropout),
            nn.Linear(config.d_model // 2, 1)
        )

    def forward(self, numerical_features, categorical_features):
        # Tokenize features
        tokens = self.tokenizer(numerical_features, categorical_features)

        # Add position embeddings
        tokens = tokens + self.pos_embedding[:, :tokens.size(1), :]

        # Transformer processing
        encoded = self.transformer(tokens)

        # Use CLS token for classification
        cls_output = encoded[:, 0, :]

        # Final prediction
        logits = self.output_head(cls_output)
        return torch.sigmoid(logits)

## 3. Conformal Prediction for Calibrated Confidence Intervals

### Research Context & Novelty

**Prior findings** lacked uncertainty quantification. **Conformal Prediction** provides distribution-free confidence intervals with guaranteed coverage, crucial for high-stakes business decisions.

**New Research (2024-2025):**
- **Majlatow et al. (2025)** demonstrate conformal prediction with probability calibration achieves 93-97% coverage guarantees in healthcare analytics [https://doi.org/10.3390/app15147925]
- **Sreenivasan et al. (2025)** show individualized diagnostic uncertainty quantification using conformal prediction enables reliable clinical decision-making [https://doi.org/10.1038/s41746-025-01616-z]
- **Shaker & Hullermeier (2025)** provide comprehensive analysis of random forest calibration, showing isotonic regression improves calibration by 15-25% [https://doi.org/10.1016/j.knosys.2025.114143]

### Implementation: Conformal Prediction Pipeline

```python
import numpy as np
from sklearn.isotonic import IsotonicRegression
from typing import Tuple, List

class ConformalPredictor:
    """Conformal prediction with split-conformal method"""
    def __init__(self, model, alpha=0.1, calibration_size=0.2):
        """
        model: base classifier with predict_proba method
        alpha: significance level (1 - coverage)
        calibration_size: proportion of data for calibration
        """
        self.model = model
        self.alpha = alpha
        self.calibration_size = calibration_size
        self.calibrator = IsotonicRegression(out_of_bounds='clip')
        self.calibration_scores = None
        self.quantile = None

    def fit(self, X_train, y_train):
        """Split data for training and calibration"""
        # Split indices
        n_cal = int(len(X_train) * self.calibration_size)
        cal_indices = np.random.choice(len(X_train), n_cal, replace=False)
        train_indices = np.setdiff1d(np.arange(len(X_train)), cal_indices)

        # Train on proper training set
        self.model.fit(X_train[train_indices], y_train[train_indices])

        # Calibrate on calibration set
        self._calibrate(X_train[cal_indices], y_train[cal_indices])

        return self

    def _calibrate(self, X_cal, y_cal):
        """Calibrate using conformal prediction"""
        # Get predicted probabilities
        probas = self.model.predict_proba(X_cal)[:, 1]

        # Compute non-conformity scores (1 - probability for true class)
        scores = []
        for i, (prob, true_label) in enumerate(zip(probas, y_cal)):
            if true_label == 1:
                score = 1 - prob  # Higher score for incorrect high confidence
            else:
                score = prob  # Higher score for incorrect low confidence
            scores.append(score)

        # Store calibration scores
        self.calibration_scores = np.array(scores)

        # Compute quantile for prediction sets
        n_cal = len(X_cal)
        self.quantile = np.quantile(
            self.calibration_scores,
            np.ceil((n_cal + 1) * (1 - self.alpha)) / n_cal
        )

    def predict_with_confidence(self, X_test) -> Tuple[np.ndarray, np.ndarray]:
        """
        Returns: (predictions, confidence_intervals)
        confidence_intervals: [lower_bound, upper_bound] for each prediction
        """
        # Get base predictions
        probas = self.model.predict_proba(X_test)[:, 1]

        # Apply conformal adjustment
        predictions = (probas >= 0.5).astype(int)

        # Compute confidence intervals
        confidence_intervals = []
        for prob in probas:
            # For class 1 prediction
            if prob >= 0.5:
                lower = max(0, prob - self.quantile)
                upper = min(1, prob + self.quantile)
            # For class 0 prediction
            else:
                lower = max(0, 1 - prob - self.quantile)
                upper = min(1, 1 - prob + self.quantile)
            confidence_intervals.append([lower, upper])

        return predictions, np.array(confidence_intervals)

    def get_prediction_sets(self, X_test, allow_empty=False) -> List[List[int]]:
        """
        Get prediction sets (multiple possible classes with coverage guarantee)
        """
        probas = self.model.predict_proba(X_test)
        prediction_sets = []

        for prob in probas:
            # Include class if 1 - probability <= quantile
            set_classes = []
            for class_idx, class_prob in enumerate(prob):
                score = 1 - class_prob
                if score <= self.quantile:
                    set_classes.append(class_idx)

            # If empty set not allowed, include most likely class
            if not allow_empty and not set_classes:
                set_classes = [np.argmax(prob)]

            prediction_sets.append(set_classes)

        return prediction_sets
