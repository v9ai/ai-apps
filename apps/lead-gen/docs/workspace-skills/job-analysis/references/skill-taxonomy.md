## Using the Taxonomy

### For Skill Extraction

When extracting skills from job descriptions:

1. Match to canonical names (e.g., "ReactJS" → "React")
2. Assign correct category
3. Note relationships (if requires React, likely needs JavaScript)

### For Skill Validation

```typescript
const taxonomy = loadSkillTaxonomy();

function isValidSkill(skillName: string): boolean {
  return taxonomy.some(
    (s) =>
      s.name.toLowerCase() === skillName.toLowerCase() ||
      s.aliases?.includes(skillName.toLowerCase()),
  );
}

function getSkillCategory(skillName: string): string {
  const skill = taxonomy.find((s) => s.name === skillName);
  return skill?.category || "unknown";
}
```

### For Recommendations

When recommending jobs, consider:

- Exact skill matches (React = React)
- Related skills (React Native if knows React)
- Complementary skills (TypeScript with JavaScript)
- Category matches (any frontend framework)

## Updating the Taxonomy

When adding new skills:

1. Choose canonical name (usually official name)
2. Assign to category
3. List common aliases
4. Note relationships with existing skills
5. Update this document
6. Increment version in SKILL.md frontmatter
