## Search Intent
Dominant intent is informational — engineers curious about how AI agents can learn from past sessions rather than starting fresh every time. Searchers want practical architecture, not theory. They're likely comparing agent frameworks (SWE-agent, Devin, OpenHands) and want to understand what "memory" means in practice.

## Recommended Structure
- **Format**: Technical deep-dive with code examples
- **Word count**: 1500-2000
- **Title tag**: "Why Do AI Agents Keep Making the Same Mistakes?"
- **Meta description**: "AI agents start fresh every session, repeating errors. We built a trajectory miner that extracts patterns from past sessions — here's the architecture, grounded in 4 research papers."
- **H1**: Why Do AI Agents Keep Making the Same Mistakes?
- **H2s**:
  1. The Stateless Agent Problem — why AI coding agents repeat mistakes
  2. Four Papers That Solved Pieces of the Puzzle — AutoRefine, ProcMEM, SWE-Replay, Beyond Static Summarization
  3. Architecture of a Trajectory Miner — pattern extraction, procedural skills, replay candidates
  4. How It Fits in a Six-Agent Pipeline — from mining to action
  5. What We Learned Building It — practical tradeoffs
